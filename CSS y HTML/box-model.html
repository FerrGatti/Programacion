Modelo de Caja en CSS

Modelo de Caja

Modelo de caja: Es la forma en la que CSS ve a los elementos HTML, tomando en cuenta sus diensiones (width & height)
Y como los ve? Como si fueran cajas con las siguientes propiedades:

1- El contenido (content): El contenido, en su mayoria texto, del elemento HTML.
2- El borde (border): Lo que delimita cada uno de los elementos HTML.
3- El relleno (padding): Son las distancias internas (la distancia del borde al contenido).
4- El margen (margin): Son las distancias externas (La distancia entre el elemento HTML y sus elementos hermanos o padres).

Dentro del modelo de caja hay que considerar que una caja tendra 4 lados:
1- Arriba (top)
2- Derecha (right)
3- Abajo (bottom)
4- Izquierda (left)

Propiedad Display

-Inline: Se comporta como un elemento en linea, puede aplicarse a elementos de bloque para cambiar sus caracteristicas
-Block: Se comporta como un elemento de bloque, puede aplicarse a elementos de linea para cambiar sus caracteristicas
-inline-block: Se comporta como un elemento de linea pero acepta modificar sus dimensiones(alto y ancho)
-None: oculta el elemento
-Table: display que viene por defecto en las tablas
-List: display que viene por defecto en las listas
-Flex: flexbox css
-Grid: Grid Css

Tamano de Caja

Utilizando el box-sizing que viene por defecto, el cual es content box, nuestra caja, a pesar de que le demos un width y un height, se vera modificada cada vez que le agreguemos padding, border, etc.
Cada px que agreguemos de padding, border, etc se sumara al tamano total de nuestra caja.
En cambio, utilizando la propiedad box sizing border, el tamano de nuestra caja se empieza a considerar desde el borde, es decir, que cada padding, cada border, etc se comienza a considerar dentro del tamano que nosotros dispongamos de la caja. Por lo cual, si le damos un height y un width, este no se modificara en el total cuando anadamos px de otras cosas, sino que reducira el tamano del contenido dependiendo de las propiedades que le vayamos agregando.

Al intentar darle margenes a nuestros arcitulos, uno pensaria que estos se suman si nosotros aplicamos margin top y margin botom, pero en lugar de esto, estos margenes se superponen haciendo que colapsen.
Por este motivo, es recomendable solo utilizar uno de los dos, ya sea margin top o margin botom.
